Architecture

ReAct Autonomous Agent Loop

Think → Act → Observe → Repeat

1. LLM receives customer messages + tool descriptions
2. LLM reasons about what to do next (Think)
3. LLM selects and calls a tool (Act)
4. System executes tool, returns result (Observe)
5. LLM decides next step → loops until FINISH
6. If AI unavailable → deterministic fallback

The LLM decides execution order, handles edge cases dynamically,
and processes each customer differently based on their intent.

Tool Registry (11 Tools)

  • parse_messages — structure raw chat
  • classify_intent — detect buy / price / complaint
  • extract_order — match against inventory
  • generate_invoice — create invoice totals
  • create_payment_task — traceable payment ref
  • create_doku_checkout — real DOKU QRIS link
  • create_approval_task — owner approval gate
  • send_telegram_approval — notify via Telegram
  • reflect_on_run — score risk
  • save_to_memory — persist for audit

Key Design Choice

Tools are deterministic. AI is only used for reasoning and tool selection. This makes the system reliable while still being autonomous.
